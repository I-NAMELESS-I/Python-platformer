# Пояснительная записка к проекту "Time Control Platformer"

## 1. Обзор архитектуры

Проект реализован на основе модульной архитектуры с разделением ответственности. Основные компоненты:

- **main.py**: Инициализация приложения, настройка окна, запуск главного цикла
- **settings.py**: Глобальные константы (пути, клавиши, размеры)
- **core/**: Реализация паттерна MVC для экранов и логики
- **entities/**: Модели игровых объектов
- **level/**: Загрузка и парсинг уровней
- **time_system/**: Специфическая механика контроля времени

## 2. Используемые библиотеки

### 2.1 Arcade
Основная графическая библиотека для 2D-игр на Python. Предоставляет:
- Отрисовку спрайтов и текста
- Физический движок (PhysicsEnginePlatformer)
- Обработку ввода (клавиатура, мышь)
- Камеры и виды (Views)
- Загрузку TMX-карт

### 2.2 Стандартные библиотеки Python
- `json`: Сохранение/загрузка настроек
- `pathlib`: Работа с путями файлов
- `dataclasses`: Структурированные данные конфигурации
- `typing`: Аннотации типов для лучшей читаемости

## 3. Описание классов

### 3.1 Core модуль

#### AppConfig
- **Назначение**: Управление настройками приложения (разрешение, полноэкранный режим, последний уровень)
- **Методы**: `load()`, `save()`, `apply_window_config()`
- **Файл**: `app_config.py`

#### Views (экраны)
- **MenuView**: Главное меню с кнопками навигации
- **GameView**: Основной игровой процесс, физика, рендеринг
- **PauseView**: Пауза с опциями продолжения/перезапуска
- **GameOverView**: Экран поражения
- **LevelCompleteView**: Экран победы
- **LevelSelectView**: Выбор уровня из списка
- **SettingsView**: Настройки видео
- **RulesView**: Просмотр правил с прокруткой

Все Views наследуются от `arcade.View` и реализуют методы `on_draw()`, `on_key_press()`, `on_mouse_press()` и т.д.

#### InputManager
- **Назначение**: Централизованная обработка ввода
- **Функции**: Преобразование клавиш в действия игрока, управление временем
- **Файл**: `input_manager.py`

#### UIWidgets
- **Button**: Кнопка с текстом и обработчиком клика
- **ButtonGroup**: Группа кнопок для управления фокусом
- **Файл**: `ui_widgets.py`

### 3.2 Entities модуль

#### Player
- **Наследование**: `arcade.Sprite`
- **Функции**: Движение, прыжки, анимации, смерть
- **Атрибуты**: Скорости, состояния, текстуры анимаций
- **Файл**: `player.py`

#### MovingPlatform
- **Назначение**: Платформы с постоянным движением в пределах границ
- **Атрибуты**: Скорость, границы движения, смещение
- **Файл**: `moving_platform.py`

#### RewindablePlatform
- **Наследование**: От MovingPlatform с дополнительными возможностями
- **Функции**: Пауза и перемотка времени
- **Файл**: `rewindable_platform.py`

### 3.3 Level модуль

#### LevelLoader
- **Назначение**: Загрузка и парсинг TMX-карт
- **Функции**: Создание спрайтов, настройка физики, инициализация объектов
- **Выход**: `LevelData` с готовыми сущностями
- **Файл**: `level_loader.py`

#### LevelData
- **Назначение**: Контейнер для загруженных данных уровня
- **Содержит**: Карту, игрока, платформы, зоны

### 3.4 Time System модуль

#### TimeSystem
- **Назначение**: Управление механикой времени
- **Функции**: Остановка времени, перемотка для выбранных объектов
- **Файл**: `time_system.py`

## 4. Особенности реализации

### 4.1 Физика
Используется `arcade.PhysicsEnginePlatformer` для реалистичного движения игрока по платформам.

### 4.2 Камера
Динамическая камера с плавным следованием за игроком и зумом.

### 4.3 Анимации
Спрайты загружаются из папок с последовательными кадрами (idle/, run/, jump/, death/).

### 4.4 Сохранение прогресса
Настройки сохраняются в JSON, последний уровень запоминается.

### 4.5 Масштабируемость
Модульная структура позволяет легко добавлять новые уровни, сущности и механики.

Проект демонстрирует хорошую организацию кода с разделением ответственности и использованием современных практик Python-разработки.</content>

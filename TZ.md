# Техническое задание (ТЗ) для проекта "Time Control Platformer"

## 1. Общие сведения

### 1.1 Название проекта
Time Control Platformer

### 1.2 Цель проекта
Разработка 2D платформера с механикой контроля времени, позволяющей игроку останавливать время и перематывать объекты назад для решения головоломок и прохождения уровней.

### 1.3 Платформа
- Операционная система: Windows
- Язык программирования: Python 3.12+
- Графическая библиотека: Arcade

## 2. Функциональные требования

### 2.1 Основной геймплей
- **Управление персонажем**: Движение влево/вправо, прыжок, быстрый спуск
- **Механика времени**:
  - Остановка времени для выбранных объектов
  - Перемотка времени назад для выбранных объектов
- **Уровни**: Загрузка уровней из TMX-файлов с тайлами, платформами, зонами смерти и финиша
- **Сущности**:
  - Игрок с анимациями (бег, прыжок, смерть)
  - Статические платформы
  - Движущиеся платформы
  - Перематываемые платформы

### 2.2 Интерфейс пользователя
- **Главное меню**: Запуск игры, выбор уровня, настройки, правила, выход
- **Игровой экран**: Камера следует за игроком, HUD отсутствует
- **Пауза**: Возможность перезапуска уровня, выхода в меню
- **Экран завершения уровня**: Поздравление с прохождением
- **Экран поражения**: Возможность перезапуска или выхода в меню
- **Выбор уровня**: Список доступных уровней
- **Настройки**: Выбор разрешения экрана, полноэкранный режим
- **Правила**: Просмотр текстовых правил с прокруткой

### 2.3 Управление
- **Клавиши**:
  - A/D: Движение влево/вправо
  - W/Space: Прыжок
  - S: Быстрый спуск
  - T: Остановка времени
  - R: Перемотка времени
  - ESC: Пауза/выход в меню
  - ЛКМ: Выбор объекта для контроля времени

## 3. Нефункциональные требования

### 3.1 Производительность
- Целевая частота кадров: 60 FPS
- Поддержка разрешений: 1280x720, 1366x768, 1600x900, 1920x1080
- Полноэкранный режим

### 3.2 Архитектура
- Модульная структура с разделением на пакеты
- Использование паттернов: MVC (Model-View-Controller), Entity-Component
- Загрузка ресурсов из внешних файлов (TMX, PNG)

### 3.3 Данные
- Сохранение настроек в JSON-файл
- Загрузка уровней из TMX-файлов (Tiled Map Editor)
- Спрайты в формате PNG с анимациями
